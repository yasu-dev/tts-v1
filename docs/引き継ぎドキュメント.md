## 1. プロジェクト概要

### 1.1 目的
災害・事故などの大規模救護現場において、トリアージタッグ情報の電子化・リアルタイム共有・搬送管理を実現するWebアプリケーション。

### 1.2 利用者
- **指揮本部（Command）**: 全体状況の把握・分析
- **トリアージ部隊（Triage）**: 現場でのタグ登録・判定
- **搬送部隊（Transport）**: 搬送指示・病院割当
- **搬送チーム（Transport Team）**: 搬送中のステータス更新
- **病院（Hospital）**: 患者受入・収容報告

### 1.3 主要機能
- QRコードスキャンによる患者登録
- STARTトリアージウィザード
- リアルタイムデータ同期
- 地図表示による位置管理
- 画像・音声記録
- 搬送管理・病院割当

詳細: [`実装済み機能一覧.md`](./実装済み機能一覧.md)

## 3. 開発環境構築

### 3.1 前提条件

- **Node.js**: 18.0以上
- **npm**: 9.0以上
- **Git**: 最新版
- **エディタ**: VS Code推奨

### 3.2 初回セットアップ

```bash
# 1. リポジトリのクローン
git clone <repository-url>
cd tts-v1

# 2. 依存パッケージのインストール
npm install

# 3. 環境変数の設定
cp .env.example .env.local
# .env.localを編集（後述）
```

### 3.3 環境変数設定

`.env.local`に以下を設定:

```bash
# Supabase設定
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# 本番環境のみ（オプション）
# NEXT_PUBLIC_ENVIRONMENT=production
```

**⚠️ 重要**:
- `.env.local`は絶対にGitにコミットしない
- 各開発者が個別に設定する
- 本番環境用の値は別途管理

### 3.4 開発サーバー起動

```bash
# 開発サーバー起動
npm run dev

# ブラウザで確認
# http://localhost:3000
```

### 3.5 ビルド確認

```bash
# プロダクションビルド
npm run build

# ビルド後の実行
npm start
```

## 7. トラブルシューティング

### 7.1 よくある問題

#### ログインできない
**症状**: ログインボタンを押してもエラー

**原因と対策**:
1. 環境変数の確認
   ```bash
   # .env.localの内容確認
   cat .env.local
   ```
2. Supabaseの接続確認
3. ユーザーが存在するか確認（Supabase Dashboard）

#### データが表示されない
**症状**: ダッシュボードにデータが表示されない

**原因と対策**:
1. RLSポリシーの確認
   ```sql
   -- 一時的に無効化してテスト
   ALTER TABLE triage_tags DISABLE ROW LEVEL SECURITY;
   ```
2. データの存在確認
3. ネットワークエラーの確認（DevTools → Network）

#### リアルタイム更新が動作しない
**症状**: データ変更が即座に反映されない

**原因と対策**:
1. Replication設定の確認
2. WebSocket接続の確認（DevTools → Network → WS）
3. ブラウザをリロード

#### 画像アップロードに失敗
**症状**: 画像アップロードでエラー

**原因と対策**:
1. Storageバケットの確認
2. ファイルサイズ確認（上限10MB）
3. ネットワーク接続確認
4. Storageの容量確認

### 7.2 エラーメッセージ一覧

| エラー | 原因 | 対策 |
|--------|------|------|
| `Failed to fetch` | API接続エラー | 環境変数、ネットワーク確認 |
| `Invalid JWT` | 認証エラー | ログアウト→再ログイン |
| `Row-level security policy violated` | RLS権限エラー | ポリシー確認 |
| `Storage bucket not found` | Storageエラー | バケット作成確認 |

### 7.3 デバッグ方法

```javascript
// console.errorは保持されているので確認可能
// ブラウザのDevToolsコンソールを確認

// Supabaseクエリのデバッグ
const { data, error } = await supabase
  .from('triage_tags')
  .select('*')

console.error('Query error:', error)
```
