# fix: モバイルレイアウトバグ 3件の修正

**確認端末:** iPad (iOS)
**対応範囲:** iOS / Android 両対応

## 1. 修正内容の詳細化

### バグ1: 一覧画面右に空白あり

**現象:** 指揮本部ダッシュボード等の一覧画面で、iPad において右側に空白ができ、横スクロールが発生する。

**原因:** `html`/`body` に `overflow-x: hidden` が設定されておらず、いずれかの子要素がビューポート幅を超過した場合に横スクロールが発生する。

**Before:** ページ全体が横スクロール可能で、右側に空白領域が表示される
**After:** 横スクロール不可。コンテンツがビューポート幅内に収まる

### バグ2: 地図が全画面でない

**現象:** 患者位置マップ（MapModal）を開いた際、Leaflet地図が画面下部まで広がらず、大きな空白が残る。

**原因:** `app/globals.css` 49-53行目のメディアクエリが、768px以下の画面で `.leaflet-container` の高さを `300px !important` に強制している。MapModal はフルスクリーンモーダルのため、この制限は不適切。

```css
/* 現在の問題コード */
@media (max-width: 768px) {
  .leaflet-container {
    height: 300px !important; /* ← MapModalでも300px固定になる */
  }
}
```

**Before:** モバイルで地図の高さが300px固定、下半分以上が空白
**After:** モーダル内の地図がヘッダーを除く全領域を使用

### バグ3: 手書きモードでツールバーが表示されない

**現象:** 現場図エディタで手書きモードに切り替えた際、画面下部に表示されるべき描画ツールバー（ペン/消しゴム/色/太さ選択）がiPadで見えない。

**原因:** DrawingToolbar は `absolute bottom-4`（底辺から16px）に配置されている。iPad（Face ID搭載モデル）ではホームインジケーター領域（safe area）が画面下部を占有するため、ツールバーがその背後に隠れる。`safe-area-inset-bottom` が考慮されていない。

**関連ファイル:** `components/scene-map/DrawingToolbar.tsx:35`

```tsx
<div className="pointer-events-auto absolute bottom-4 left-1/2 z-30 -translate-x-1/2">
```

**Before:** 描画ツールバーがホームインジケーター / ナビゲーションバーの背後に隠れて操作不能
**After:** safe area を考慮し、常にアクセス可能な位置に表示される

## 2. 実現手段

### バグ1: 一覧画面右に空白

| 対象ファイル      | 変更方針                                                 |
| ----------------- | -------------------------------------------------------- |
| `app/globals.css` | `@layer base` 内に `body { overflow-x: hidden; }` を追加 |

**方針:** body 全体で横方向のオーバーフローを抑止する。`html` に設定するとスクロール挙動に副作用が出る可能性があるため、`body` のみに設定する。

### バグ2: 地図が全画面でない

| 対象ファイル            | 変更方針                                                                             |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `app/globals.css:49-53` | モバイル用 `.leaflet-container { height: 300px !important; }` のメディアクエリを削除 |

**方針:** この300px制限はMapModal導入以前にインライン地図用に追加されたものと推測される。現在の地図表示はすべてモーダル内で行われており、親コンテナの `h-full` / `flex-1` で高さが制御されるため、この固定値は不要。削除しても他の地図表示に影響しない（TriageMap は親から `h-full` で高さを受け取る設計）。

### バグ3: 手書きモードでツールバーが表示されない

| 対象ファイル                                   | 変更方針                                                       |
| ---------------------------------------------- | -------------------------------------------------------------- |
| `components/scene-map/DrawingToolbar.tsx:35`   | `bottom-4` → `bottom-14` に変更                                |
| `components/scene-map/SceneMapEditor.tsx:1271` | スタンプモードインジケーターも `bottom-4` → `bottom-14` に変更 |

**方針:** bottom オフセットを 16px → 56px に拡大し、モバイル端末のシステムUI領域を確実に回避する。

**`bottom-14` (56px) の採用理由:**

- iPad ホームインジケーター (~34px) を超える
- Android ジェスチャーナビゲーション (~48px) を超える
- デスクトップでも許容範囲の位置（画面下部から56px）
- `viewport-fit=cover` 不要 → 他ページのヘッダーにレグレッションが発生しない
- プラットフォーム非依存の CSS → iOS / Android / デスクトップ全てで動作

**`viewport-fit=cover` + `env(safe-area-inset-bottom)` を不採用とした理由:**

- `viewport-fit=cover` はコンテンツをセーフエリア外に拡張するため、全ダッシュボードのヘッダーがステータスバー背後に回り込むレグレッションが発生する
- 全ヘッダーに `safe-area-inset-top` 対応を追加する必要があり、修正範囲が拡大する

**新規ファイル:** なし

## 4. 類似箇所の横展開

### スタンプモード表示バー

`SceneMapEditor.tsx:1271` のスタンプモードインジケーターも `absolute bottom-4` を使用している。

```tsx
{paletteSelection && !isDrawing && (
  <div className="pointer-events-auto absolute bottom-4 left-1/2 z-30 -translate-x-1/2">
```

**対応:** DrawingToolbar と同じ `bottom-14` に変更する。

### 他のフローティングUI

`SceneMapEditor.tsx:1227` のフローティングパレットは `bottom-16` を使用している。safe-area を加味しても十分な余裕がある（64px + safe-area）。変更不要。

### 他のモーダルの地図表示

現在、地図を表示するモーダルは MapModal のみ。SceneMapModal は Konva Canvas を使用しており、Leaflet の 300px 制限の影響を受けない。横展開不要。

### 他のダッシュボードの overflow-x

`body` レベルで `overflow-x: hidden` を設定するため、全ダッシュボードに一括で効果が及ぶ。個別対応は不要。
