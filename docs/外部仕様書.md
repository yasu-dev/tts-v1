## 1. システム概要

### 1.1 システムの目的

災害・事故などの大規模救護現場において、トリアージタッグ情報を電子化し、リアルタイムで共有・管理するWebアプリケーション。

### 1.2 利用者とロール

| ロール | 名称 | 主な業務 |
|--------|------|---------|
| IC | 指揮本部 | 全体状況把握、搬送指示、分析 |
| TRI | トリアージ部隊 | 現場でのタグ登録、トリアージ判定 |
| TRN | 搬送部隊 | 搬送先病院の割当、搬送指示 |
| Transport Team | 搬送チーム | 搬送中のステータス更新 |
| HSP | 病院 | 患者受入、収容報告 |

### 1.3 アクセス方法

- **URL**: https://your-domain.com
- **対応ブラウザ**:
  - Chrome 90以上
  - Safari 14以上
  - Edge 90以上
- **対応デバイス**:
  - PC（推奨: デスクトップ画面）
  - タブレット（推奨: iPad以上）
  - スマートフォン

### 2.3 トリアージ部隊画面 (`/triage/scan`)

#### 画面構成
```
┌─────────────────────────────────────┐
│ [ QRスキャナー表示エリア ]            │
├─────────────────────────────────────┤
│ または                               │
│ [ 手動でタグ番号入力 ]                │
│                                      │
│ [ 患者登録フォーム ]                  │
│  ・バイタルサイン                     │
│  ・患者情報                          │
│  ・画像アップロード                   │
│  ・音声入力                          │
│                                      │
│ [ 登録 ]                             │
└─────────────────────────────────────┘
```

#### 機能

##### QRスキャン
- カメラ起動
- QRコード読取
- タグ番号自動入力

##### STARTウィザード
4段階の質問形式:
1. 「歩けますか？」→ はい/いいえ
2. 「呼吸していますか？」→ はい/いいえ
3. 「脈が触れますか？」→ はい/いいえ
4. 「意識はありますか？」→ はい/いいえ

結果: 自動的にトリアージ区分を判定（赤・黄・緑・黒）

##### バイタルサイン入力（1st/2nd/3rd）
| 項目 | 形式 | 単位 |
|------|------|------|
| 判定者名 | テキスト | - |
| 判定場所 | テキスト | - |
| 判定時間 | 時刻 | HH:MM |
| 意識 | 選択 | I/II/III |
| 呼吸 | 数値 | 回/分 |
| 脈拍 | 数値 | 回/分 |
| 血圧（収縮期/拡張期） | 数値 | mmHg |
| 体温 | 数値 | °C |

##### 患者情報入力
| 項目 | 必須 | 形式 |
|------|------|------|
| 氏名 | ❌ | テキスト |
| 電話番号 | ❌ | テキスト |
| 住所 | ❌ | テキスト |
| 年齢 | ❌ | 数値 |
| 性別 | ❌ | 選択（男性/女性/不明） |

##### 症状・傷病名（複数選択可）
- 打撲
- 骨折
- 捻挫
- 切断
- 熱傷
- その他（自由記述）

##### 画像アップロード
- **最大枚数**: 15枚
- **対応形式**: JPEG, PNG
- **最大サイズ**: 10MB/枚
- **自動圧縮**: 1920px以下にリサイズ、80%品質
- **画像タイプ**: 外傷部位/現場写真/身体図/その他

##### 音声入力
- 主訴入力
- 特記事項入力
- ブラウザの音声認識API使用

#### ボタン
- **登録**: データを保存してダッシュボードへ戻る

## 3. 機能仕様

### 3.1 認証機能

#### ログイン
- **方式**: Supabase Auth（メール＋パスワード）
- **セッション**: ブラウザに自動保存
- **有効期限**: 7日間（デフォルト）
- **自動ログイン**: セッション有効時は自動ログイン

#### ログアウト
- 各ダッシュボード右上の「ログアウト」ボタン
- セッション削除
- ログイン画面へリダイレクト

#### アクセス制御
- 未認証時: ログイン画面へ自動リダイレクト
- ロール別アクセス制御: RLSポリシーで実装

---

### 3.2 リアルタイム更新機能

#### 仕組み
- Supabase Realtimeを使用
- WebSocket接続
- データベース変更を即座に反映

#### 対象テーブル
- `triage_tags`: 患者情報の変更
- `hospitals`: 病院情報の変更

#### 更新インジケーター
- データ更新時に「データ更新」メッセージ表示
- 2秒間表示後に自動非表示

---

### 3.3 QRスキャン機能

#### 対応デバイス
- スマートフォン（カメラ必須）
- タブレット（カメラ必須）
- PC（Webカメラ必須）

#### 動作
1. 「QRスキャン」ボタンクリック
2. カメラ権限要求
3. カメラ起動
4. QRコード読取
5. データ解析
6. 該当患者を特定
7. 次のアクション実行

#### エラー処理
- カメラ権限なし: 「カメラ権限が必要です」
- QRコード認識失敗: 「QRコードの読み取りに失敗しました」
- 患者未登録: 「該当する患者が見つかりません」

---

### 3.4 画像管理機能

#### アップロード
- **方式**: Supabase Storageに直接アップロード
- **圧縮**: 自動リサイズ（最大1920px、80%品質）
- **進捗表示**: アップロード中のプログレスバー
- **エラーハンドリング**: ネットワークエラー時の再試行

#### 保存先
- **Bucket**: `triage-images`
- **パス**: `{tagId}/{timestamp}_{index}.jpg`

#### 表示
- サムネイル表示（患者詳細モーダル）
- クリックで拡大表示
- 画像タイプの表示（バッジ）

---

### 3.5 音声入力機能

#### 対応ブラウザ
- Chrome（推奨）
- Edge
- Safari（制限あり）

#### 動作
1. マイクボタンクリック
2. マイク権限要求
3. 音声認識開始
4. テキスト変換
5. 入力フィールドに追記

#### エラー処理
- マイク権限なし: 「マイク権限が必要です」
- 認識失敗: 「音声認識に失敗しました」

## 5. 操作フロー

### 5.1 患者登録フロー（トリアージ部隊）

```
1. ログイン（tri@demo.com）
    ↓
2. /triage/scan に自動遷移
    ↓
3. QRスキャンまたは手動入力でタグ番号取得
    ↓
4. STARTウィザードでトリアージ判定
    ↓
5. バイタルサイン入力（1st）
    ↓
6. 患者情報入力
    ↓
7. 症状・傷病名選択
    ↓
8. 画像アップロード（任意）
    ↓
9. 音声入力（任意）
    ↓
10. 「登録」ボタンクリック
    ↓
11. データ保存＆ダッシュボードへ
```

### 5.2 搬送フロー（搬送部隊→搬送チーム→病院）

```
【搬送部隊（TRN）】
1. ログイン（trn@demo.com）
    ↓
2. /transport に自動遷移
    ↓
3. 患者を選択
    ↓
4. 病院・救急隊を選択
    ↓
5. 「搬送開始」クリック
    ↓
6. ステータスが「現場」→「病院準備」に変更

【搬送チーム（Transport Team）】
7. ログイン（チームアカウント）
    ↓
8. /transport-team に自動遷移
    ↓
9. 割当患者を確認
    ↓
10. QRスキャンまたは「更新」ボタン
    ↓
11. ステータスが「病院準備」→「病院へ」に変更
    ↓
12. 病院到着後、再度更新
    ↓
13. ステータスが「病院へ」→「病院」に変更

【病院（HSP）】
14. ログイン（hsp@demo.com）
    ↓
15. /hospital に自動遷移
    ↓
16. 搬送中患者リストに表示
    ↓
17. 「受入」ボタンクリック
    ↓
18. ステータスが「病院」に確定
    ↓
19. リストから自動削除
```

## 6. エラー処理

### 6.1 ユーザー向けエラーメッセージ

| 状況 | メッセージ | 対処方法 |
|------|-----------|---------|
| ログイン失敗 | メールアドレスまたはパスワードが正しくありません | 再入力 |
| ネットワークエラー | ネットワークエラーが発生しました | 再試行 |
| 権限エラー | この操作を実行する権限がありません | 管理者に連絡 |
| データ保存失敗 | 更新に失敗しました | 再試行 |
| 画像アップロード失敗 | {N}枚の画像をアップロードできませんでした | ネットワーク確認 |
| カメラ権限なし | カメラ権限が必要です | ブラウザ設定で許可 |
| QRスキャン失敗 | QRコードの読み取りに失敗しました | 再試行 |

### 6.2 エラーログ

- `console.error`で記録
- ブラウザのDevToolsコンソールで確認可能
- Supabase Logsでサーバー側エラーを確認

## 7. 性能要件

### 7.1 レスポンス時間

| 操作 | 目標時間 |
|------|---------|
| ページ読み込み | 3秒以内 |
| ログイン | 2秒以内 |
| データ保存 | 2秒以内 |
| リアルタイム更新 | 1秒以内 |
| 画像アップロード（1枚） | 5秒以内 |

- **想定**: 最大100ユーザー同時接続

| 項目 | 制限 |
|------|------|
| 画像ファイル | 10MB/枚 |
| Storage総容量 | 1GB（Free Plan） |
| Database | 500MB（Free Plan） |
