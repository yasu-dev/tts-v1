## 1. 事前準備

### 1.1 必要なツール

以下のツールをインストールしてください：

#### Node.js（必須）
- **バージョン**: 18.0以上
- **ダウンロード**: https://nodejs.org/
- **確認方法**:
  ```bash
  node --version
  # v18.0.0 以上であればOK
  ```

#### npm（必須）
- Node.jsと同時にインストールされます
- **確認方法**:
  ```bash
  npm --version
  # 9.0.0 以上であればOK
  ```

#### Git（必須）
- **ダウンロード**: https://git-scm.com/
- **確認方法**:
  ```bash
  git --version
  # git version 2.x.x 以上であればOK
  ```

#### VS Code（推奨）
- **ダウンロード**: https://code.visualstudio.com/
- **推奨拡張機能**:
  - ESLint
  - Prettier
  - TypeScript and JavaScript Language Features

### 1.2 アカウント準備

#### Supabaseアカウント（必須）
1. https://supabase.com/ にアクセス
2. 「Start your project」をクリック
3. GitHubアカウントでサインアップ（推奨）
4. アカウント作成完了

#### GitHubアカウント（推奨）
- リポジトリアクセスのため
- https://github.com/ でアカウント作成

## 2. ローカル開発環境構築

### 2.1 リポジトリのクローン

```bash
# 1. 作業ディレクトリに移動
cd ~/Documents  # Macの場合
cd C:\Users\YourName\Documents  # Windowsの場合

# 2. リポジトリをクローン
git clone <repository-url>

# 3. プロジェクトディレクトリに移動
cd tts-v1

# 4. ディレクトリ構造を確認
ls -la  # Mac/Linux
dir  # Windows
```

**期待される出力**:
```
app/
components/
docs/
lib/
public/
supabase/
package.json
README.md
...
```

### 2.2 依存パッケージのインストール

```bash
# npmを使用してインストール
npm install

# ⏱️ 所要時間: 約3-5分
```

**期待される出力**:
```
added XXX packages, and audited XXX packages in XXs
```

**エラーが発生した場合**:
```bash
# キャッシュをクリアして再試行
npm cache clean --force
npm install
```

### 2.3 環境変数の設定

```bash
# 1. サンプルファイルをコピー
cp .env.example .env.local  # Mac/Linux
copy .env.example .env.local  # Windows

# 2. .env.localを編集
# VS Codeで開く場合:
code .env.local
```

`.env.local`の内容を以下のように編集:
```bash
# Supabase設定（後で値を設定）
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**⚠️ 重要**:
- この時点では値が空でOK
- 次のステップ（Supabase設定）で値を取得します

## 3. Supabase設定

### 3.1 新しいプロジェクトを作成

1. https://supabase.com/dashboard にアクセス
2. 「New Project」をクリック
3. 以下の情報を入力:
   - **Project name**: triage-tag-system（任意）
   - **Database Password**: 強力なパスワードを設定
     - 例: `Tr1@geT@g2025!`
     - **必ず記録してください**
   - **Region**: Northeast Asia (Tokyo) を選択
4. 「Create new project」をクリック
5. ⏱️ 約2-3分待つ（プロジェクトの初期化）

### 3.2 APIキーの取得

プロジェクト作成後:

1. 左サイドバー → **Settings** → **API** をクリック
2. 以下の値をコピー:
   - **Project URL**
     ```
     https://xxxxx.supabase.co
     ```
   - **anon public** key
     ```
     eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...（非常に長い文字列）
     ```

3. `.env.local`に貼り付け:
   ```bash
   NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

4. **保存**

### 3.3 データベースのセットアップ

#### ステップ1: SQL Editorを開く

1. 左サイドバー → **SQL Editor** をクリック
2. 「New query」をクリック

#### ステップ2: SQLスクリプトを実行

**重要**: 以下の順序で実行してください

##### (1) テーブルスキーマの拡張

```bash
# docs/database/add-triage-tag-fields.sql の内容をコピー
```

1. ファイル `docs/database/add-triage-tag-fields.sql` を開く
2. 全内容をコピー
3. SQL Editorに貼り付け
4. 「Run」をクリック
5. ✅ "Success. No rows returned" が表示されればOK

##### (2) データベース関数の定義

```bash
# docs/database/contact-point-functions.sql の内容をコピー
```

同様の手順で実行

##### (3) セキュリティポリシーの設定

```bash
# docs/database/rls-policies.sql の内容をコピー
```

同様の手順で実行

**⚠️ 必須**: RLSポリシーは必ず設定してください

##### (4) デモユーザーの作成（開発環境のみ）

**推奨方法**: Supabase Dashboard UI

1. 左サイドバー → **Authentication** → **Users** をクリック
2. 「Add user」をクリック
3. 以下のユーザーを1人ずつ作成:

| Email | Password | その他設定 |
|-------|---------|-----------|
| ic@demo.com | password | ✅ Auto Confirm User |
| tri@demo.com | password | ✅ Auto Confirm User |
| trn@demo.com | password | ✅ Auto Confirm User |
| hsp@demo.com | password | ✅ Auto Confirm User |

**または**: SQLで一括作成
```bash
# docs/database/create-demo-users.sql の内容をコピー
# SQL Editorで実行
```

##### (5) デモデータの投入（開発環境のみ）

```bash
# docs/database/demo-data.sql の内容をコピー
# SQL Editorで実行
```

### 3.4 Realtimeの有効化

1. 左サイドバー → **Database** → **Replication** をクリック
2. 以下のテーブルを有効化:
   - ✅ `triage_tags`: 「0 tables」をクリック → `triage_tags`を選択
   - ✅ `hospitals`: 同様に選択
3. 「Save」をクリック

### 3.5 Storageの設定

1. 左サイドバー → **Storage** をクリック
2. 「Create a new bucket」をクリック
3. 以下を入力:
   - **Name**: `triage-images`
   - **Public bucket**: ✅ チェック
   - **File size limit**: `10485760`（10MB）
4. 「Create bucket」をクリック

### 3.6 認証設定（開発環境用）

1. 左サイドバー → **Authentication** → **Settings** をクリック
2. **Email Auth**セクション:
   - ❌ **Enable email confirmations**: OFFにする（開発時のみ）
   - ❌ **Enable email change confirmations**: OFFにする（開発時のみ）
3. 「Save」をクリック

**⚠️ 重要**: 本番環境では必ずONにしてください

## 5. トラブルシューティング

### 5.1 よくある問題

#### Q1: `npm install`でエラーが出る

**症状**:
```
npm ERR! code ERESOLVE
npm ERR! ERESOLVE unable to resolve dependency tree
```

**対策**:
```bash
# 1. node_modulesを削除
rm -rf node_modules  # Mac/Linux
rmdir /s node_modules  # Windows

# 2. package-lock.jsonを削除
rm package-lock.json  # Mac/Linux
del package-lock.json  # Windows

# 3. 再インストール
npm install
```

#### Q2: ログインできない

**症状**: 「メールアドレスまたはパスワードが正しくありません」

**確認事項**:
1. `.env.local`の設定を確認
   ```bash
   cat .env.local  # Mac/Linux
   type .env.local  # Windows
   ```
2. Supabase Dashboard → Authentication → Users でユーザーが存在するか確認
3. Email確認が無効化されているか確認

**対策**:
```bash
# 開発サーバーを再起動
# Ctrl+C で停止 → npm run dev で再起動
```

#### Q3: データが表示されない

**症状**: ダッシュボードにデータが表示されない

**確認事項**:
1. デモデータが投入されているか確認
   - Supabase Dashboard → Table Editor → `triage_tags`
2. RLSポリシーが設定されているか確認
3. ブラウザのコンソールでエラーを確認
   - F12キー → Console タブ

**一時的な対策**:
```sql
-- RLSを一時的に無効化（テストのみ）
ALTER TABLE triage_tags DISABLE ROW LEVEL SECURITY;

-- データ確認後、必ず再有効化
ALTER TABLE triage_tags ENABLE ROW LEVEL SECURITY;
```

#### Q4: Realtimeが動作しない

**症状**: データを変更しても画面に反映されない

**確認事項**:
1. Replicationが有効化されているか確認
2. ブラウザのDevTools → Network → WS タブで WebSocket接続を確認
3. コンソールにエラーが出ていないか確認

**対策**:
```bash
# ブラウザをリロード（Ctrl+R または Cmd+R）
```

#### Q5: 画像アップロードに失敗

**症状**: 「画像をアップロードできませんでした」

**確認事項**:
1. Storageバケット `triage-images` が作成されているか
2. バケットがPublicになっているか
3. ファイルサイズが10MB以下か
4. ネットワーク接続を確認

### 5.2 開発サーバーが起動しない

**症状**:
```
Error: Cannot find module 'next'
```

**対策**:
```bash
# 依存パッケージを再インストール
npm install
```

**症状**:
```
Port 3000 is already in use
```

**対策**:
```bash
# 既存のプロセスを停止
# Mac/Linux:
lsof -ti:3000 | xargs kill

# Windows:
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```

### 5.3 TypeScriptエラー

**症状**:
```
Type error: Property 'xxx' does not exist on type 'yyy'
```

**対策**:
```bash
# 型チェック
npx tsc --noEmit

# エラー箇所を確認して修正
```

### 5.4 Supabase接続エラー

**症状**:
```
Failed to fetch
```

**確認事項**:
1. `.env.local`の値を再確認
2. Supabaseプロジェクトが起動しているか確認
3. ネットワーク接続を確認

**対策**:
```bash
# 環境変数を再読み込み
# 開発サーバーを再起動
```
